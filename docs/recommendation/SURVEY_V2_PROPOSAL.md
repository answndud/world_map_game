# survey v2 개정안 초안

## 목적

이 문서는 현재 `survey-v2 / engine-v2` 추천 엔진을 14개 페르소나 시나리오로 평가한 뒤, 다음 개선에서 어떤 부분을 먼저 고쳐야 하는지 정리한 초안이다.

이번 문서의 목표는 바로 코드를 바꾸는 것이 아니라, `어디가 약한지 -> 무엇을 먼저 바꿀지 -> 어떤 순서로 실험할지`를 명확히 만드는 것이다.

## 현재 baseline에서 우선 개선이 필요한 시나리오

### 1. `P04`

- 기대: `독일, 덴마크, 캐나다`
- 실제: `우루과이, 칠레, 스페인`
- 문제 해석:
  - `복지`보다 `기후/물가/균형형 생활 리듬`이 더 크게 작용한다.
  - welfare 우선 사용자가 기대하는 “공공 시스템의 성숙도”보다 `중간 물가 + 온화한 기후` 쪽이 강하게 보인다.

### 2. `P06`

- 기대: `포르투갈, 스페인, 체코`
- 실제: `우루과이, 아일랜드, 말레이시아`
- 문제 해석:
  - `LOW budget` 제약이 아직 충분히 강하지 않다.
  - 영어 친화도와 안전 점수가 일부 비싼 국가를 너무 오래 상위권에 남긴다.

### 3. `P13`

- 기대: `영국, 캐나다, 네덜란드`
- 실제: `미국, 싱가포르, 아랍에미리트`
- 문제 해석:
  - `FAST + CITY + HIGH english + DIVERSITY` 조합에서 영어/도시성 점수가 너무 강해, `MILD climate` 선호가 약해진다.
  - 그래서 온화한 기후를 원해도 뜨거운 초도시형 국가가 상단에 남는다.

## v2 제안 원칙

현재는 질문 수를 8개로 늘린 상태이므로, 다음 개선은 문항 수를 더 늘리기보다 아래 원칙으로 간다.

1. 먼저 현재 8문항 구조를 유지한다.
2. helper text와 선택지 설명을 더 분명하게 다듬는다.
3. 가중치와 penalty를 먼저 조정한다.
4. 그래도 개선이 부족하면 그때 새 문항 추가를 검토한다.

이렇게 해야 기존 만족도 데이터와 비교도 쉽고, 설문 복잡도도 급격히 늘어나지 않는다.

## v2 제안 1. `engine-v2` 가중치 실험

### A. climate mismatch penalty 추가

현재는 `거리 2 이상`에서 점수가 0이 될 뿐, 별도 패널티는 없다.

그래서 `MILD` 선호인데도 `SGP`, `ARE` 같은 뜨거운 초도시형 국가가 영어/도시성 덕분에 상단에 남을 수 있다.

제안:

- 기후 거리 2 이상이면 작은 penalty 추가
- 특히 `MILD` 선호 사용자는 극단 기후 국가가 오래 남지 않도록 조정

### B. 영어 점수 비중 재조정

현재 `HIGH -> englishSupport * 4`, `MEDIUM -> * 2`는 일부 시나리오에서 영향이 크다.

제안:

- `HIGH`를 조금 낮추거나
- `MEDIUM`을 더 낮춰 “영어가 있으면 좋다”가 전체 순위를 너무 흔들지 않게 한다

### C. 저예산 초과 패널티 강화

`LOW budget` 사용자는 사실상 제약이 강하다.

제안:

- `actualPriceLevel > preferredPriceLevel`일 때 penalty를 더 키운다
- 특히 차이가 2 이상이면 더 강한 penalty를 준다

## v2 제안 2. `survey-v2` 문구 개정

질문 수는 유지하되, 아래 문구를 더 또렷하게 바꾼다.

### A. budget helper text 강화

현재 `LOW / MEDIUM / HIGH`는 이해는 쉽지만 “비싸면 불편한가, 아니면 감수 가능한가”가 조금 흐릴 수 있다.

제안:

- `LOW`: 생활비 제약이 커서 비싼 나라면 추천에서 확실히 밀려야 한다
- `MEDIUM`: 장점이 뚜렷하면 어느 정도 감수 가능하다
- `HIGH`: 생활비보다 인프라와 환경이 더 중요하다

### B. welfare 라벨 정밀화

`복지`는 추상적으로 읽힐 수 있다.

제안:

- `복지` 대신 `복지 / 공공 서비스`
- helper text도 “사회 안전망, 공공 시스템, 장기 정착 안정성”을 더 분명히 적는다

## v2 목표

다음 실험의 목표는 간단하다.

- baseline 11/14 -> 최소 12/14
- `P04`, `P06`, `P13` 중 최소 2개 개선
- `P01`, `P02`, `P14` 같은 강한 시나리오는 유지

즉, 기존 강점을 망가뜨리지 않고 약한 시나리오만 먼저 끌어올리는 것이 목표다.

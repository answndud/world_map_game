# 추천 설문 오프라인 평가 시나리오

이 문서는 WorldMap 추천 기능을 개발 단계에서 품질 평가할 때 쓰는 시나리오 모음이다.

런타임 LLM 호출은 하지 않고, 서브 에이전트나 사람이 이 표를 기준으로 설문 문항, 선택지, 가중치, 추천 결과 품질을 반복 개선한다.

평가 기준은 두 가지다.

1. 서버 추천 결과가 페르소나의 취향과 맞는가
2. 만족도 점수를 받았을 때 설문 버전과 엔진 버전을 개선할 근거가 충분한가

답변값은 `RecommendationSurveyAnswers` enum 이름을 그대로 쓴다.

현재 baseline은 두 층으로 나뉜다.

1. 기존 14개 시나리오
   - 새로 추가한 `정착 성향`, `이동 생활 방식` 두 문항을 모두 `BALANCED`로 둔다.
   - 기존 baseline을 과하게 흔들지 않기 위한 기준선이다.
2. 추가 4개 시나리오
   - 새 두 문항을 적극적으로 써서 top 3가 실제로 달라지는지 확인한다.
   - 같은 기본 취향에서도 `EXPERIENCE / TRANSIT_FIRST`와 `STABILITY / SPACE_FIRST`가 후보 구성을 어떻게 바꾸는지 비교한다.

## 읽는 방법

이 표는 정답 목록이 아니라 `오프라인 품질 평가 기준`이다.

어떤 시나리오는 결과 상위 1~3위가 정확히 일치하지 않아도 괜찮다.

중요한 것은 아래다.

1. 추천 결과가 페르소나의 핵심 의도를 잘 잡는가
2. 너무 비슷한 후보만 반복적으로 나오지 않는가
3. 만족도 점수가 낮게 나오는 시나리오를 통해 설문 문항이나 가중치를 개선할 수 있는가

## 운영 메모

- `surveyVersion`을 바꾸면 이 표도 같이 갱신한다.
- `engineVersion`을 바꾸면 기대 후보와 기대 만족도 범위를 다시 점검한다.
- 새 설문 문항을 추가할 때는 최소 2개 이상의 페르소나를 추가한다.
- 낮은 만족도 시나리오는 삭제하지 말고, 설문 개선용 실패 케이스로 남긴다.

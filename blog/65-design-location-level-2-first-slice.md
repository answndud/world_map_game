# 위치 찾기 Level 2 첫 조각을 어떻게 열지 설계하기

## 왜 바로 구현부터 하지 않았는가

위치 찾기 게임은 이미 Level 1에서 복잡도가 높다.

- 3D 지구본 렌더링
- 국가 선택 판정
- 하트 기반 재시도
- Stage / Attempt 저장
- endless run

여기서 Level 2를 생각나는 대로 붙이기 시작하면,
타이머, 힌트, 전체 194개 국가, 소국, streak bonus가 한 번에 섞여 버린다.

그러면 구현은 할 수 있어도
왜 그렇게 만들었는지 설명하기 어려워진다.

그래서 이번 단계는
코드를 늘리기 전에
`첫 번째 작은 조각을 어디까지로 자를 것인가`
를 먼저 고정하는 설계 단계다.

## 이번 글의 핵심 질문

`위치 찾기 Level 2를 처음 열 때,
Level 1과 충분히 다르면서도 지금 구조를 깨지 않는 첫 조각은 무엇인가?`

내 결론은 이렇다.

- 입력 방식은 그대로 둔다
- `세션 / Stage / Attempt / 하트` 구조도 그대로 둔다
- 대신 `오답 시 거리 + 방향 힌트`를 서버가 계산해 내려준다

즉,
첫 조각은 새 게임을 만드는 것이 아니라
`같은 게임 루프 위에 더 높은 난도와 힌트 정책을 올리는 것`
이다.

## 왜 이 방향이 맞는가

### 1. Level 1 구조를 그대로 재사용할 수 있다

위치 게임은 이미 아래 구조가 있다.

- `LocationGameSession`
- `LocationGameStage`
- `LocationGameAttempt`
- 하트 3개
- 정답 시 다음 Stage
- 오답 시 같은 Stage 재시도

이 구조가 이미 설명 가능하게 잡혀 있기 때문에,
Level 2도 여기 위에서 시작하는 것이 가장 자연스럽다.

### 2. 프론트 리스크를 크게 늘리지 않는다

처음부터 타이머나 좌표 입력으로 가면
지구본 상호작용을 다시 크게 흔들 수 있다.

반면 이번 첫 조각은

- 선택 방식은 그대로 두고
- 서버가 힌트 payload를 추가로 내려주는 방식

이라서 리스크가 더 작다.

### 3. 9단계 목표와도 맞다

플레이북 9단계에는

- 힌트
- 부분 점수
- 오차 거리 표시

가 들어 있다.

위치 게임 Level 2를 `거리 + 방향 힌트`
중심으로 여는 것은 이 목표와도 잘 맞는다.

## Level 1과 무엇이 달라지나

### Level 1

- 주요 국가 중심
- 오답이면 하트만 감소
- 정답/오답만 즉시 확인

### Level 2 첫 조각

- 더 헷갈리는 국가 조합을 우선 출제
- 오답이면 하트 감소
- 동시에 서버가 `거리(km)`와 `방향` 힌트 제공
- 점수는 나중에 `hint debt` 반영 가능

즉,
Level 2는 더 어려운 문제를 주되,
오답 피드백은 더 설명적으로 만든다.

## 왜 힌트 계산은 서버가 해야 하는가

거리와 방향은 프론트에서도 계산할 수 있어 보인다.

하지만 그렇게 하면 안 된다.

이유는 세 가지다.

1. 정답 기준 정보는 서버가 이미 가진다
2. 어떤 힌트를 언제 줄지도 게임 규칙이다
3. 나중에 점수 감점과 랭킹 설명까지 연결하려면
   힌트와 판정이 같은 쪽에 있어야 한다

그래서 첫 조각에서는

- `LocationGameLevel2HintPolicy`
- 또는 `LocationGameDistanceHintService`

처럼 별도 정책 클래스로 분리하는 것이 맞다.

## 요청 흐름은 어떻게 잡을까

첫 조각 기준 흐름은 이렇게 잡는다.

1. `POST /api/games/location/sessions`
   - `gameLevel=LEVEL_2`
2. `GET /state`
   - 현재 Stage, 점수, 하트, level
3. `POST /answer`
   - 국가 선택 제출
4. 서버 판정
5. 오답이면
   - `distanceKm`
   - `directionHint`
   를 응답에 같이 담는다

즉,
프론트는 힌트를 그려주고,
힌트 생성 규칙은 서버가 가진다.

## 첫 조각에서 일부러 하지 않을 것

이번 설계에서 일부러 뒤로 미룬 것은 아래다.

- 194개 전체 자산 전면 확대
- 타이머
- streak bonus
- 소국 / 영토 모드
- 카메라 자동 줌 힌트
- SSE

첫 조각 범위를 넘기면
설명 가능성이 바로 떨어진다.

## 구현 순서

실제 구현은 이 순서가 맞다.

1. `LocationGameLevel` enum 추가
2. 세션 시작 시 level 저장
3. Level 2 출제 정책 추가
4. 오답 응답 payload에 `distanceKm`, `directionHint` 추가
5. play 화면 오버레이에 힌트 노출
6. 결과 화면에 힌트 사용 흔적을 남길지 결정

## 면접에서는 이렇게 설명하면 된다

위치 찾기 Level 2는 Level 1을 다시 만드는 것이 아니라,
이미 있는 `세션 / Stage / Attempt / 하트` 구조 위에
난도 정책과 힌트 정책만 추가하는 방향으로 설계했습니다.
첫 조각은 타이머보다 `오답 시 거리/방향 힌트`를 서버가 계산해 내려주는 방식으로 시작해서,
프론트 리스크를 키우지 않으면서도 Level 1과 분명히 다른 규칙을 만들 계획입니다.

# [Spring Boot 포트폴리오] 22. 게스트 플레이를 유지하면서 단순 계정으로 확장하는 설계

## 이번 글의 핵심 질문

지금 WorldMap은 가입 없이 바로 플레이할 수 있다.

이 장점은 유지하고 싶다.

하지만 동시에:

- 점수를 계속 누적하고 싶고
- 내 기록을 마이페이지에서 보고 싶고
- 로그인한 계정에 랭킹과 전적이 쌓이면 좋겠다

이 요구도 있다.

그러면 질문은 하나다.

“비회원은 지금처럼 세션 기반으로 바로 플레이하게 두면서, 기록을 유지하고 싶은 사람만 가볍게 로그인하게 만들 수 있을까?”

이번 단계에서는 이걸 위한 단순 계정 설계를 먼저 고정했다.

## 왜 복잡한 계정 시스템으로 가지 않는가

이 프로젝트의 계정 목적은 커뮤니티가 아니다.

핵심은 아래뿐이다.

- 닉네임 유지
- 점수 누적
- 내 랭킹 확인
- 내 최근 플레이 보기

그래서 MVP에서는 아래를 빼기로 했다.

- 이메일 인증
- 비밀번호 찾기
- 친구
- 팔로우
- 프로필 이미지
- 댓글 / SNS 기능

대신 `닉네임 + 비밀번호`만으로 끝나는 단순 계정 구조를 택한다.

## 설계의 핵심

### 1. 비회원은 그대로 세션 기반

- 가입 없이 바로 플레이
- 현재 브라우저 세션 안에서는 흐름 유지
- 기록을 길게 보존할 책임은 지지 않음

### 2. 회원은 기록 보존용

- 닉네임으로 가입
- 비밀번호 해시 저장
- 로그인 후 내 전적과 내 랭킹 확인

즉, 계정은 “서비스 이용 자격”보다 “기록 보존 수단”에 가깝다.

### 3. 비회원 기록은 현재 브라우저 세션 안에서만 계정으로 귀속

이게 가장 중요한 포인트다.

회원가입 또는 로그인 시, 현재 세션의 `guestSessionKey`를 가진 미귀속 기록만 그 계정에 붙인다.

그래서 구현이 단순해진다.

## 데이터 모델은 어떻게 바뀌는가

계정 테이블은 최대한 단순하게 간다.

### `member`

- `id`
- `nickname`
- `passwordHash`
- `role`
- `createdAt`
- `lastLoginAt`

닉네임이 곧 로그인 ID이자 공개 이름이다.

### 기존 게임 / 랭킹 레코드

게임 세션과 랭킹 레코드에는 아래 둘 중 하나만 채운다.

- `memberId`
- `guestSessionKey`

그리고 항상 `nicknameSnapshot`은 남긴다.

이렇게 해야 과거 기록과 랭킹 표시는 당시 이름 기준으로 안정적이다.

## 요청 흐름은 어떻게 될까

### 회원가입

1. 닉네임/비밀번호 제출
2. `member` 저장
3. 로그인 세션 생성
4. 현재 `guestSessionKey` 기록을 새 계정으로 귀속
5. `/mypage` 이동

### 로그인

1. 닉네임/비밀번호 확인
2. 로그인 세션 생성
3. 현재 `guestSessionKey` 기록 귀속
4. `/mypage` 이동

### 로그아웃

1. 로그인 세션 해제
2. 새 비회원 플레이를 위한 새 `guestSessionKey` 발급

## 왜 이 설계가 포트폴리오에 좋은가

이 구조는 화려하지는 않지만 설명이 아주 쉽다.

- 왜 비회원 플레이를 유지했는가
- 왜 이메일 없이 단순 계정으로 시작했는가
- 왜 `memberId`와 `guestSessionKey`를 같이 두는가
- 왜 비회원 기록 귀속 범위를 “현재 브라우저 세션”으로 제한했는가

모두 서비스 목적과 자연스럽게 연결된다.

## 다음 구현 순서

1. `member` 엔티티 추가
2. `guestSessionKey` 유틸 추가
3. 게임/랭킹 레코드에 `memberId`, `guestSessionKey` 연결
4. 회원가입 / 로그인 / 로그아웃 추가
5. `/mypage` 실제 데이터 연결
6. admin 접근 제어 추가

## 면접에서는 이렇게 설명하면 된다

“이 서비스는 커뮤니티가 아니라 점수 누적형 게임 허브라서, 비회원은 그대로 세션 기반으로 바로 플레이하게 두고 기록을 오래 남기고 싶은 사람만 단순 계정으로 로그인하게 설계했습니다. 계정은 `닉네임 + 비밀번호`만 두고, 게임과 랭킹 레코드는 `memberId` 또는 `guestSessionKey` 둘 중 하나로 소유자를 식별합니다. 로그인 시에는 현재 브라우저 세션의 비회원 기록만 계정으로 귀속해 구조를 단순하게 유지했습니다.”

# [Spring Boot 포트폴리오] 38. 홈 첫 화면에서 모드 중복 노출을 걷어내고 진입 구조 단순화하기

## 왜 이 글을 쓰는가

홈 첫 화면은 사용자가 가장 먼저 보는 곳인데, 한동안 같은 정보가 여러 번 반복되고 있었다.

예를 들어 이런 구조였다.

- hero에서 `위치 미션 시작`, `인구수 퀴즈 시작`, `나라 추천 받기`, `랭킹 보기`
- 오른쪽 패널에서도 모드 이름 4개를 다시 보여 줌
- 본문 `모드 구성` 카드에서도 같은 모드 이름을 또 보여 줌

즉, “무엇을 할 수 있는가”는 분명했지만, 정보가 겹치면서 오히려 첫 화면이 복잡해졌다.

이번 조각의 목표는 홈에서 모드 이름을 한 번만 보여 주고, hero는 서비스 소개와 계정 / 통계 진입만 맡게 정리하는 것이었다.

## 이번에 바꾼 핵심

- hero에서는 개별 게임 CTA를 모두 제거했다.
- hero 버튼은 `모드 선택하기`, `서비스 현황 보기`만 남겼다.
- 실제 모드 선택은 `지금 플레이할 모드` 카드 영역에서만 하게 만들었다.
- 하단 설명 패널도 `처음 시작한다면`, `기록은 이렇게 이어집니다`처럼 역할이 분명한 구조로 다시 썼다.

즉, 홈은 “서비스를 소개하는 곳”이고, 실제 모드 선택은 “카드 영역 한 곳”에서만 하도록 책임을 나눴다.

## 어떤 파일이 바뀌었는가

- `src/main/java/com/worldmap/web/HomeController.java`
- `src/main/resources/templates/home.html`
- `src/main/resources/static/css/site.css`
- `src/test/java/com/worldmap/web/HomeControllerTest.java`

## 요청 흐름

이번에도 흐름 자체는 단순하다.

```text
GET /
-> HomeController
-> modeCards + entrySteps + accountNotes model 구성
-> home.html SSR 렌더링
```

중요한 점은 홈이 상태를 바꾸지 않는다는 것이다.

- guest/member 분기: `home.html`이 현재 session을 직접 읽음
- 로그인/로그아웃: 기존 auth 흐름 재사용
- 게임 시작: 각 모드 카드 링크가 기존 game 시작 페이지로 이동

즉, 홈은 진입 구조만 정리하고 실제 상태 변화는 계속 기존 계층이 맡는다.

## 왜 hero에서 직접 게임 시작 버튼을 걷어냈는가

처음에는 “바로 시작”을 강조하려고 hero에 게임 버튼을 많이 넣었다.

하지만 실제로는 아래 문제가 생겼다.

- 모드 이름을 여러 번 읽게 됨
- 어디가 대표 진입점인지 흐려짐
- 오른쪽 패널과 카드 영역 역할이 겹침

그래서 지금은 이렇게 정리했다.

- hero: 서비스 소개, 계정 연결, 공개 `Stats` 진입
- mode cards: 실제 모드 선택
- 하단 패널: 시작 흐름과 기록 흐름 설명

이렇게 나누면 홈에서 “무엇을 먼저 눌러야 하는지”가 더 분명해진다.

## 왜 controller에 더 많은 상태를 넣지 않았는가

이번 작업은 도메인 로직 추가가 아니라 public shell 정리다.

그래서 `HomeController`는 아래 정도만 만든다.

- `modeCards`
- `entrySteps`
- `accountNotes`

반대로 아래는 여전히 템플릿이나 기존 계층이 맡는다.

- guest/member 인증 상태 분기: `home.html`
- 로그아웃 처리: `AuthPageController`
- 실제 모드 시작: 각 game/recommendation/ranking 컨트롤러

이렇게 해야 홈 컨트롤러가 “첫 화면 설명용 read model” 역할에 머물고, auth나 game 책임을 가져오지 않는다.

## 테스트는 무엇을 했는가

- `HomeControllerTest`
  - `지금 플레이할 모드`, `기록은 이렇게 이어집니다`, `Stats`, `모드 선택하기`가 보이는지
  - 예전 중복 문구였던 `위치 미션 시작`, `인구수 퀴즈 시작`, `모드 구성`, `플레이 방식`이 더 이상 보이지 않는지
  - ADMIN 세션에서는 `Dashboard`가 계속 보이는지

핵심은 “새 문구가 보이는가”보다 “불필요한 중복이 실제로 사라졌는가”를 같이 고정한 것이다.

## 회고

홈은 기능을 많이 보여 주는 곳이 아니라, 사용자가 가장 자연스럽게 다음 행동으로 넘어가게 만드는 곳이다.

이번 조각은 새 기능을 붙인 것이 아니라, 이미 있던 기능을 더 설명 가능하고 깔끔한 구조로 다시 배치한 작업에 가깝다.

이런 정리는 작은 템플릿 수정처럼 보여도, 실제 서비스 인상과 사용 흐름에는 큰 차이를 만든다.

## 면접에서는 이렇게 설명할 수 있다

“처음 홈 화면에는 같은 모드 이름이 hero, 요약 패널, 카드에서 반복 노출돼서 구조가 복잡했습니다. 그래서 hero에서는 서비스 소개와 계정/Stats 진입만 남기고, 실제 모드 선택은 카드 영역 한 곳에서만 하도록 책임을 다시 나눴습니다. 요청 흐름은 그대로 `HomeController -> home.html`이고, 홈은 상태를 바꾸지 않고 SSR에서 진입 구조를 어떻게 보여 줄지만 맡게 했습니다.”

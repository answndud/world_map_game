# Work Log Entry Template

Use this structure when appending a new entry to `/Users/alex/project/worldmap/docs/WORKLOG.md`.

```md
## YYYY-MM-DD - 작업 이름

- 단계:
- 목적:
- 변경 파일:
- 요청 흐름:
- 데이터 / 상태 변화:
- 핵심 도메인 개념:
- 예외 / 엣지 케이스:
- 테스트:
- 배운 점:
- 아직 약한 부분:
- 면접용 30초 요약:
```

## Writing Rules

- Prefer concrete files over vague descriptions.
- Describe request flow, not just file edits.
- If no tests ran, write `없음` or explain why.
- Keep `아직 약한 부분` honest and specific.
- The 30-second summary should sound like a spoken interview answer.

## Minimum Checklist

- What changed?
- Why now?
- Where does the request start and where does state change?
- What edge case matters?
- What test proves the change?
- What do I still not fully understand?
